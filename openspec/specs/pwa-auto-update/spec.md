# PWA Auto-Update

Service worker configuration for auto-updating the PWA in the background.

## ADDED Requirements

### Requirement: Service worker is registered

The service worker SHALL be registered on application load to enable PWA capabilities.

#### Scenario: Service worker registered on app load
- **WHEN** the application loads in a browser that supports service workers
- **THEN** the service worker script at `/sw.js` (or generated by vite-plugin-pwa) SHALL be registered

#### Scenario: Service worker registration fails gracefully
- **WHEN** the browser does not support service workers or registration fails
- **THEN** the application SHALL continue to function normally without service worker features

### Requirement: Auto-update via skipWaiting

The service worker SHALL use skipWaiting to activate immediately when a new version is available.

#### Scenario: New service worker activates immediately
- **WHEN** a new service worker is installed and takes over
- **THEN** the new service worker SHALL call skipWaiting() to activate without waiting for old tabs to close

#### Scenario: Existing clients are claimed
- **WHEN** a new service worker becomes active
- **THEN** it SHALL call clientsClaim() to take control of all existing client pages immediately

### Requirement: Static assets cached with stale-while-revalidate

Static assets (JavaScript, CSS, images, fonts) SHALL be cached using a stale-while-revalidate strategy.

#### Scenario: Asset served from cache immediately
- **WHEN** a user requests a static asset
- **THEN** the service worker SHALL respond immediately with the cached version if available

#### Scenario: Cache updated in background
- **WHEN** a cached asset is served
- **THEN** the service worker SHALL simultaneously fetch the latest version from the network and update the cache for subsequent requests

### Requirement: Navigation requests use NetworkFirst

Navigation requests (HTML pages) SHALL use a NetworkFirst strategy to ensure users get fresh content.

#### Scenario: Fresh HTML fetched on navigation
- **WHEN** a user navigates to a new page
- **THEN** the service worker SHALL attempt to fetch the HTML from the network first

#### Scenario: Fallback to cached HTML when offline
- **WHEN** the user is offline and requests a page
- **THEN** the service worker SHALL serve the cached HTML if available, otherwise return an appropriate error

### Requirement: PWA manifest is complete

The web app manifest SHALL contain all required fields for proper PWA installation on iOS, Android, and desktop.

#### Scenario: Manifest contains required fields
- **WHEN** the browser requests the manifest
- **THEN** it SHALL include:
  - `name`: "Coffee-Diary"
  - `short_name`: "Coffee-Diary"
  - `start_url`: "/"
  - `scope`: "/"
  - Valid icon references

### Requirement: Auto-update works across platforms

The auto-update mechanism SHALL work on all platforms that support service workers.

#### Scenario: Update applied on iOS
- **WHEN** a new version is deployed and the user opens the PWA on iOS
- **THEN** the new version SHALL be loaded automatically

#### Scenario: Update applied on Android
- **WHEN** a new version is deployed and the user opens the PWA on Android
- **THEN** the new version SHALL be loaded automatically

#### Scenario: Update applied on desktop
- **WHEN** a new version is deployed and the user opens the PWA in a desktop browser
- **THEN** the new version SHALL be loaded automatically
